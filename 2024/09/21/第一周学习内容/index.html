待解答的问题：

问题一：

目录符号都有哪些？用途是啥？

在 Linux 系统中的常用目录符号:

  1. .（点号）：

    • 代表当前目录。
    • 用途：在命令中可以用来指定当前目录。例如，./script.sh 表示在当前目录下执行 script.sh 文件。

    2...（两个点号）：

    • 代表上级目录。
    • 用途：可以用来返回到当前目录的上一级。例如，cd .. 命令将当前目录切换到其上级目录。

    3.~（波浪号）：

    • 代表当前用户的主目录。
    • 用途：用于快速访问用户的主目录。例如，cd ~ 会切换到当前用户的主目录，~/Documents 表示主目录下的 Documents 文件夹。

    3./（斜杠）：代表根目录。

    • 用途：用于构建绝对路径。例如，/etc/hosts 表示从根目录开始的绝对路径。

    4.-（连字符）：

    • 在某些命令中，- 用作标志，表示标准输入或标准输出。
    • 用途：例如，cat - 表示从标准输入读取数据。
    • 另外，在某些命令中（如 cd），- 用来切换到上一个目录。例如，cd - 会带你回到你之前所在的目录。

问题二：

什么是绝对目录？什么是相对目录？

  • 绝对路径是从系统根目录开始的完整路径，无论当前的工作目录是什么，都能准确定位文件位置。

  • 相对路径所“相对”的对象是工作目录，不用从根目录开始写起，相较于绝对路径不那么冗长。

  • 两个路径在使用场景上也有所不同：

    • 绝对路径主要用来确保路径唯一且不受当前工作目录的影响，也就是说，无论在哪个工作目录内，文件都可以正常的调用
    • 相对路径可以用来方便在同一个项目/目录结构内工作

补充问题：

当不能使用cat和tac时如何读取Linux的文件

1.more filename 分页显示文件内容，按空格键来翻页，按 q 键退出查看

2.less filename，类似于more，但是更加灵活，按空格键向前翻一整页，按b键向后翻一整页，同样可以按b退出

5.grep（Global Regular Expression Print），其中Regular Expression正是大名鼎鼎的反人类的正则表达式

grep命令是一个非常强大的文本搜索工具，用于在文件或输入流中查找匹配特定模式的行，同时支持正则表达式。

  • 基本用法：grep 选项 模式 文件

模式：要搜索的文本模式，普通字符串或正则表达式

文件：要搜索的一个或多个文件。如果不指定文件，grep 将从标准输入中读取数据（可以通过管道传入数据）。

  • 常用选项：**-i**：忽略大小写

    **-v**：显示不匹配的行。

    **-r 或 -R**：递归地搜索目录中的文件。

    **-n**：显示匹配行的行号。

    **-l**：只显示包含匹配模式的文件名。

    **-w**：只匹配整个单词。

​ **-o**：只输出匹配的部分。

​ **-c**：只输出匹配的行数。

​ ** -A**：在匹配的行后面显示指定行数的内容。

​ ** -B**：在匹配的行前面显示指定行数的内容。

​ **-C**：在匹配的行前后各显示指定行数的内容。

HTTP

问题一：HTTP是什么？

HTTP（Hypertext Transfer Protocol，超文本传输协议）是一种用于传输网页数据的协议。它是互联网中的主要通信协议，定义了客户端（如浏览器）和服务器之间如何交换数据。HTTP 是无状态的，这意味着每个请求都是独立的，不会记住之前的请求状态。HTTP 通常使用 TCP 作为传输层协议，运行在 80 端口上（在 HTTPS 中则使用 443 端口）。

问题二：URL是什么？

URL（Uniform Resource Locator，统一资源定位符）是指向互联网资源的地址。它提供了一个资源的具体位置和访问方法。一个典型的 URL 包含多个部分，如下所示：

  • 协议（如 http:// 或 https://）：指定使用的协议。
  • 主机名（如 www.example.com）：指定服务器的域名或 IP 地址。
  • 端口（可选，如 :80 或 :443）：指定服务器上的端口，HTTP 默认使用端口 80，HTTPS 默认使用端口 443。
  • 路径（如 /path/to/resource）：指定服务器上的资源位置。
  • 查询字符串（可选，如 ?key=value&key2=value2）：用于传递参数给服务器。
  • 片段标识符（可选，如 #section1）：指定资源中的某一部分。

问题三：HTTP请求包的结构有哪几部分组成？

HTTP 请求包的结构通常包括以下几部分：

  1. 请求行：包含请求方法（如 GET、POST）、请求 URI 和 HTTP 版本。例如：GET /index.html HTTP/1.1。
  2. 请求头：包含各种头部信息，用于提供关于请求的附加信息，如 Host、User-Agent、Accept 等。
  3. 空行：请求头和请求体之间的空行，用于分隔请求头和请求体。
  4. 请求体（可选）：在某些请求方法（如 POST、PUT）中，包含发送给服务器的数据。在 GET 请求中通常没有请求体。

问题四：常见的请求头有哪些？它们有什么作用？

常见的 HTTP 请求头包括：

  1. Host：指定请求的主机名和端口号，通常用于虚拟主机。比如 Host: www.example.com。
  2. User-Agent：指定发起请求的客户端软件信息，如浏览器类型和版本。例如：User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/91.0.4472.124 Safari/537.36。
  3. Accept：指定客户端能够处理的内容类型。例如：Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,*/*;q=0.8。
  4. Accept-Language：指定客户端能够处理的语言类型。例如：Accept-Language: en-US,en;q=0.5。
  5. Accept-Encoding：指定客户端能够处理的内容编码方式，如压缩方式。例如：Accept-Encoding: gzip, deflate, br。
  6. Connection：指定是否保持连接的持续性。例如：Connection: keep-alive。
  7. Authorization：用于传递认证信息，如令牌或凭据。例如：Authorization: Bearer <token>。
  8. Referer:指示当前请求的来源页面,帮助服务器了解用户是从哪个页面点击过来的
  9. Cookie:传递客户端存储的 Cookie 数据给服务器,包含客户端存储的 Cookie 数据
  10. Content-Type:指定请求体的媒体类型,帮助服务器正确解析请求体数据。例如，application/json 表示请求体包含 JSON 数据，而 application/x-www-form-urlencoded 表示表单数据。这个头部对于 POST 和 PUT 请求尤为重要，因为它们通常包含请求体。
  11. X-Forwarded-For:用于标识客户端的原始 IP 地址

问题五：请求方式有哪些？GET和POST有什么区别？如何通过GET和POST传参？

HTTP 请求方式（方法）包括：

  • GET：用于请求资源。请求的数据附加在 URL 中，适合获取数据，不改变服务器状态。数据通过查询字符串传递，例如：GET /search?Welcome_to=ROIS。
  • POST：用于提交数据给服务器，通常用于创建或更新资源。请求的数据放在请求体中，不暴露在 URL 中，例如：POST /submit，请求体中包含数据。

GET 和 POST 的区别：

  1. 数据传递：

    GET：数据附加在 URL 的查询字符串中，长度有限制（约 2KB），对敏感数据不安全。

    POST：数据放在请求体中，没有长度限制，更适合传递大量数据和敏感信息。

  2. 缓存

    GET 请求通常可以被缓存，浏览器可能会缓存响应。

    POST 请求通常不会被缓存。

  3. 安全性：

    GET 请求的参数在 URL 中明文传递，不适合传递敏感数据。

    POST 请求的数据在请求体中，相对更安全，但仍需 HTTPS 加密传输以确保安全性。

通过 GET 和 POST 传参：

GET 传参：数据通过 URL 的查询字符串传递，例如 http://yuanshen.com/search?a=op。

POST 传参：数据通过请求体传递，可以使用表单数据、JSON 格式或其他编码方式。例如，表单数据可以用 application/x-www-form-urlencoded 编码，JSON 数据用 application/json 编码。(这个格式在做Special Http这道题的时候就有出现过)

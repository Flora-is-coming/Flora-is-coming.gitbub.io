一些知识的缺漏：

空合并运算符 ??

用于简化对变量的检查，特别是在处理可能未定义或为 null 的变量时。

语法：

  • **如果 $value1 存在且不为 null**，则 $variable 将被赋值为 $value1。
  • **如果 $value1 不存在或为 null**，则 $variable 将被赋值为 $value2。

这个运算符在留言板中可以检查用户输入是否为空

如果用户不输入任何值就提交表单，那么$_POST['message']的值将为NULL

实现思路

  1. 先写一个用于登录的前端页面(login.html)，利用 表单输入用户名和密码。还有一个注册按钮，点了会跳转到注册的前端页面(register.HTML)。
  2. 注册前端页面(register.HTML)也是用表单提交用户名和密码，提交后的数据被传到register.php进行后续的操作
  3. register.php要连接数据库，把用户名和密码都插入admin的数据表里面，返回注册成功/失败的消息，若成功就跳转回登录页面(login.html)
  4. 登录页面(login.html)，上传用户名和密码到login.php，进行验证身份的操作，若成功验证就跳转到留言页面（message.html)
  5. 留言页面（message.html)，有表单来提交留言，有一个提交按钮来确认。把提交的留言和用户名上传到message.php进行下一步操作
  6. message.php，用于把留言上传到mes_container数据表里
  7. 在message.html中显示所有留言

数据库中要有两个数据表admin和mes_container。

  • 表单所必须的元素name，这个名称会成为数据提交时的键，用户输入的值则成为对应的值。例如，提交用户表单的name属性值为username,那么在后端就可以用$_POST['username']，来访问所提交的值。

  • JavaScript Window Location:

    首先，window 是浏览器中的全局对象，代表浏览器的窗口或框架。

    其次，location 是 window 对象的一个属性，代表当前文档的 URL。

    它同时也是一个 Location 对象，包含有关当前文档的各个部分（如协议、主机名、路径等）的信息。

    用于获得当前页面的地址 (URL)，并把浏览器重定向到新的页面。Location的两个常用属性如下

    • window.location.assign(url) ： 加载 URL 指定的新的 HTML 文档。 就相当于一个链接，跳转到指定的url，当前页面会转为新页面内容，可以点击后退返回上一个页面。

    • window.location.replace(url) ： 通过加载 URL 指定的文档来替换当前文档 ，这个方法是替换当前窗口页面，前后两个页面共用一个窗口，所以是没有后退返回上一页的
